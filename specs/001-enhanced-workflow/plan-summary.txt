KEY TECHNICAL DECISIONS
================================================================================

Threading vs Multiprocessing
  DECISION: Threading (not multiprocessing)
  Rationale: FFmpeg runs as subprocess (releases GIL), shared AppState is simpler,
             lower memory overhead per worker

Drag-Drop Implementation
  DECISION: Hybrid approach (tkinterdnd2 + file dialog fallback)
  Rationale: Best UX when available, graceful degradation without hard dependency

Template Profile Storage
  DECISION: Store full profile settings inline (not by reference)
  Rationale: Templates dont break if original profile deleted, full independence

GPU Encoder Fallback
  DECISION: Automatic CPU fallback on GPU OOM or unavailability
  Rationale: Ensures batch completes even if GPU fails mid-processing

Filter Ordering
  DECISION: Fixed order (rotate-crop-scale-color-deinterlace-delogo)
  Rationale: Predictable results, avoid user confusion, prevents artifacts

Parallel Worker Count
  DECISION: Default to (cpu_cores - 1) / 2, capped at 4, user-adjustable
  Rationale: Balances performance with system responsiveness, prevents overload

QUICK REFERENCE: COMMAND EXAMPLES
================================================================================

Detect FFmpeg encoders:
  ffmpeg -encoders

Extract video metadata:
  ffprobe -v error \
    -show_entries format=duration,size \
    -show_entries stream=width,height,codec_name,bit_rate,r_frame_rate \
    -of json \
    input.mp4

Build FFmpeg command with filters:
  ffmpeg -i input.mp4 \
    -vf "rotate=PI/2,crop=1920:800:0:140,scale=1280:720,eq=brightness=0.2" \
    -c:v libx264 -preset fast -crf 23 \
    output.mp4

GPU encoding with NVIDIA NVENC:
  ffmpeg -i input.mp4 \
    -c:v h264_nvenc -preset fast -rc vbr -cq 23 \
    output.mp4
